浮点数学习笔记

1. 什么是浮点数

浮点数就是用来表示：
- 小数
- 很大或很小的数

比如：

3.14
0.5
1000000.0
0.000001

在 C / C++ 里常见的浮点类型有：

float
double
long double

其中最常见的是：
- float
- double


2. 为什么叫“浮点”

因为小数点的位置不是固定的。

比如十进制里：

123.45
1.2345 × 10^2
0.12345 × 10^3

本质上是同一个数，只是小数点“浮动”了。

计算机里的浮点数也是类似思想，只不过底数通常不是 10，而是 2。


3. 浮点数最大的问题：它通常不是精确的

这是最最重要的一句：

浮点数很多时候只是近似值，不是精确值。

比如你以为：

0.1

就是“精确的 0.1”。

但在二进制里，很多十进制小数根本不能有限表示，所以计算机只能存一个很接近它的值。

就像十进制里：

1 / 3 = 0.333333333...

你永远写不完。

同样地，在二进制里，0.1 这种数也可能是“写不完”的，所以只能截断。


4. 所以会发生什么现象

你可能会看到这种情况：

double a = 0.1 + 0.2;
double b = 0.3;

你以为 a == b 应该是 true。

但实际上不一定。

因为：
- 0.1 是近似存储
- 0.2 是近似存储
- 0.3 也是近似存储

它们最后的二进制表示可能不完全一样。

所以：

a == b

不一定成立。


5. 浮点数和整数的区别

整数比较“硬”。

比如：

2 + 3 = 5

只要没溢出，就是精确的。

但是浮点数不是这样。浮点数的优点是：
- 能表示小数
- 能表示范围很大的数

代价就是：
- 很多值不精确
- 运算过程中会有舍入误差

所以你可以这样记：
- 整数：精确，但范围和形式更受限制
- 浮点：灵活，但经常近似


6. IEEE-754 是什么

现代计算机里，浮点数通常遵循一种标准，叫：

IEEE-754

你不用一开始就死记细节，但要知道：

一个浮点数通常分成三部分：
- 符号位 sign
- 指数 exponent
- 尾数 mantissa / significand

你可以粗略理解成：

值 = 符号 × 有效数字 × 2^指数


7. 单精度 float 的结构

常见的 float 一般是 32 位：
- 1 位符号位
- 8 位指数
- 23 位尾数

意思是：
- 1 位决定正负
- 一部分决定数量级有多大
- 一部分决定具体数字长什么样

所以浮点数并不是“直接把小数原样塞进内存”。

它是用一种科学计数法式的结构来存。


8. 为什么说它像科学计数法

比如十进制里你会写：

1234.5 = 1.2345 × 10^3

浮点数也是类似想法，只不过它通常是二进制：

1.xxxxx × 2^n

所以它特别适合表示：
- 很大
- 很小

的数。

这也是浮点数范围很强的原因。


9. 特殊值：inf 和 NaN

浮点数不只是“普通数字”。

它还能表示一些特殊值：

1）正无穷和负无穷

比如结果太大，超出可表示范围时，可能得到：

inf
-inf

2）NaN

意思是：

Not a Number

比如某些非法结果可能得到 NaN，例如：

0.0 / 0.0

所以浮点数世界里，不是只有普通数字。


10. 为什么不能随便用 == 比较浮点数

因为 == 比较的是：

两个值是不是完全一模一样

但浮点数在计算过程中常常会有一点点误差。

所以很多时候，你真正想问的不是：

它们是不是完全相同？

而是：

它们是不是足够接近？

这两个问题不是一回事。


11. 更合理的比较方法

常见写法是：

fabs(a - b) < eps

意思是：

如果两个数的差的绝对值小于某个很小的范围，就认为它们“差不多相等”。

比如：

if (fabs(a - b) < 1e-9)
{
    // treat as equal
}

但是这里还有一个更深的点：

固定 eps 也不一定总是对。

因为：
- 对很大的数来说，1e-9 太小了
- 对很小的数来说，1e-9 又可能太大了

所以以后你会学到“绝对误差”和“相对误差”这类更合理的比较方式。

你现在先记住：

浮点数比较相等，不要默认用 ==。


12. epsilon 是什么

你可以把 epsilon 理解成：

浮点数能分辨的最小间隔的一种代表

文章里的定义是：

最小的 x，使得 1 + x > 1

这说明在 1 附近，浮点数不是无限细的。

它只能分辨到某个粒度。

这个概念会帮助你理解：
- 为什么浮点数不精确
- 为什么比较相等会麻烦
- 为什么小误差会存在


13. 一个很危险的问题：中间步骤溢出

有时候最后答案本来在范围内，但中间计算先炸了。

比如：

double x = 1e200;
double y = sqrt(x * x + x * x);

理论上这个式子没问题。

但中间的：

x * x

可能已经太大了，先变成 inf。

这说明一件很重要的事：

写数学公式时，不能只看最终答案，还要看中间步骤会不会出问题。

这在图形学、数值计算、几何里很常见。


14. 另一个大坑：相近的大数相减

这个问题叫：
- loss of significance
- catastrophic cancellation

意思是：

两个很接近的大数相减时，前面很多有效位抵消掉了，最后留下来的结果精度很差。

比如你本来两个数都挺大、挺精确，但它们一减，结果反而变得不可靠。

你现在不用死磕数学细节，但一定要记住：

“两个很接近的浮点数相减” 是危险操作。


15. 打印出来的样子，不等于它真实的值

浮点数显示的时候，你看到的只是“打印格式”处理后的结果。

比如：
- %f
- %e

打印风格不同，输出长相不同。

但这不表示底层存储真的不同。

也就是说：

显示层和存储层是两回事。

这点很重要，因为初学者很容易以为“打印出来像 0.3，就是真的精确 0.3”。

不是这样的。


16. 为什么 float 和 double 不一样

一般来说：
- double 比 float 更精确
- double 也通常能表示更大的范围

所以很多时候如果你不确定，用 double 会更稳一点。

你可以先粗略记成：
- float：精度较低
- double：精度较高


17. 写代码时你最该有的直觉

以后看到这种代码：

double a = 0.1 + 0.2;
double b = 0.3;

你脑子里要自动弹出一句话：

“这两个数不一定完全相等。”

看到这种：

sqrt(x * x + y * y)

你脑子里要想：

“如果 x 和 y 特别大，中间会不会先溢出？”

看到这种：

big1 - big2

如果两个数特别接近，你要警觉：

“这里会不会丢失有效数字？”

这就是浮点数思维。


18. 给你一份超短版总结

如果你只想记最核心的内容，就记这几句：

1）
浮点数通常是近似值，不是精确值。

2）
很多十进制小数不能被二进制精确表示。

3）
浮点数一般由：
符号位 + 指数 + 尾数
组成。

4）
不要轻易用 == 比较浮点数。

5）
数学上等价的公式，计算机里数值表现不一定一样。

6）
打印出来的样子不等于底层真实存储方式。


19. 如果你是为了 C / C++ 学这些，你当前够用了什么程度

如果你现在只是为了写 C / C++、做学校项目、看懂基础资料，那你目前先做到下面这几点就已经很够用了：

- 知道 float / double 是近似数
- 知道浮点数比较不能太天真
- 知道会有 inf 和 NaN
- 知道运算顺序和公式写法会影响结果
- 知道打印只是显示，不是底层真相

这就已经比很多初学者强很多了。


20. 最后送你一句最值得记的话

浮点数不是错，它只是“有限位数下对实数的近似表示”。

所以遇到奇怪结果，不要第一反应觉得“电脑坏了”或“编译器疯了”。

很多时候，是因为你现在开始接触到：

计算机并不会真的存“数学上的实数”，它只是在模拟。
